## 1. 项目概述

### 1.1 目标
开发一个Edge浏览器扩展，提供划词翻译功能，支持单词和句子的本地大模型翻译、音标显示、语音朗读。

### 1.2 核心功能
- **划词翻译**: 选中页面文字后自动识别单词或句子
- **智能展示**: 单词用悬浮弹窗，句子用侧边栏面板
- **多语言支持**: 可配置多种源语言和目标语言
- **本地模型**: 支持Ollama、vLLM、LMStudio、LiteLLM等本地大模型
- **语音朗读**: 使用浏览器Web Speech API朗读原文和译文
- **音标显示**: 单词翻译提供音标信息

## 3. 功能详细设计

### 3.1 划词检测

**触发方式**:
- 鼠标释放时检测是否有选中文本
- 支持快捷键触发（可配置，默认Alt+T）

**文本分类**:
- **单词**: 选中文本长度 < 20字符，且不含空格
- **句子**: 选中文本长度 ≥ 20字符，或包含空格

**防误触机制**:
- 划词长度需 > 2字符才触发
- 排除输入框内的选中文本
- 可配置黑名单网站

### 3.2 悬浮弹窗（单词）

**布局**:
```
┌─────────────────────────────┐
│  📢  original               │  ← 原文 + 朗读按钮
│  /ɔˈrɪdʒənl/                │  ← 音标
├─────────────────────────────┤
│  原文                        │  ← 译文
│  📢                          │  ← 译文朗读按钮
└─────────────────────────────┘
```

**行为**:
- 跟随鼠标位置显示（智能定位避免超出视口）
- 点击外部区域自动关闭
- 支持拖拽移动位置
- ESC键关闭

### 3.3 侧边栏（句子）

**实现方式**: 使用 **Chrome Side Panel API** (chrome.sidePanel)
- 理由：提供原生侧边栏体验，支持跨标签页保持状态
- Fallback：如API不可用，使用content script注入的DOM覆盖层

**布局**:
```
┌──────────────────────────────────────────────────────────────────┐
│  原网页内容                              │  翻译侧边栏            │
│                                          │  ┌────────────────┐   │
│                                          │  │  翻译结果        │   │
│                                          │  │  [原文] 📢      │   │
│                                          │  │  选中的句子...   │   │
│                                          │  │                │   │
│                                          │  │  [译文] 📢      │   │
│                                          │  │  翻译后的句子... │   │
│                                          │  │                │   │
│                                          │  │  [复制] [关闭]   │   │
│                                          │  └────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

**行为**:
- 固定宽度300px，右侧滑入
- 可调整宽度
- 点击关闭按钮或ESC键关闭
- 复制按钮可复制译文
- 权限要求: `sidePanel` permission in manifest.json

### 3.4 LLM引擎适配

**接口策略**: 
- 优先使用OpenAI兼容格式 (`/v1/chat/completions`)
- Ollama同时支持原生格式 (`/api/generate`) 作为fallback
- 所有引擎统一封装为`chat.completions.create()`接口

**支持的引擎**:

| 引擎 | 默认地址 | API格式 | 特点 |
|------|----------|---------|------|
| Ollama | http://localhost:11434 | `/v1/chat/completions` 或 `/api/generate` | 本地运行，模型丰富 |
| vLLM | http://localhost:8000 | `/v1/chat/completions` | 高性能推理 |
| LMStudio | http://localhost:1234 | `/v1/chat/completions` | 图形化管理 |
| LiteLLM | 可配置 | `/v1/chat/completions` | 统一多模型网关 |

**翻译Prompt模板**:

**单词翻译**:
```
请翻译以下单词，并提供音标。
格式要求：
- 音标：/音标/
- 释义：中文释义
- 例句：一个英文例句（中文翻译）

单词：{word}
目标语言：{targetLanguage}
```

**句子翻译**:
```
请将以下文本翻译成{targetLanguage}，保持原文的语气和风格。

原文：{text}

翻译：
```

**响应解析策略**:
- **JSON模式**: 优先使用模型的JSON输出能力，要求按固定格式返回
- **Fallback**: 非JSON响应使用正则表达式提取关键信息
- **音标提取**: 从响应中匹配 `/.../` 格式的IPA音标
- **容错处理**: 如果LLM未按格式返回，直接显示原始响应

### 3.5 语音朗读(TTS)

**实现**: 使用Web Speech API (`speechSynthesis`)

**功能**:
- 朗读原文和译文
- 可配置语音（语速、音调、音量）
- 支持中途停止

**语音选择**:
- 自动选择匹配语言的语音
- 用户可在设置中手动选择优先语音

## 4. 配置设计

### 4.1 设置页面配置项

**基础设置**:
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 引擎类型 | 下拉框 | Ollama | Ollama/vLLM/LMStudio/LiteLLM |
| API地址 | 文本 | http://localhost:11434 | 根据引擎自动建议 |
| 模型名称 | 文本 | qwen2.5:7b | 使用的模型 |

**语言设置**:
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 源语言 | 多选 | ["英文"] | 可选：英文、日文、韩文、法文、德文、俄文等 |
| 目标语言 | 单选 | 中文 | 可选：中文、英文、日文等 |

**语音设置**:
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 原文语音 | 下拉框 | 自动 | 朗读原文使用的语音 |
| 译文语音 | 下拉框 | 自动 | 朗读译文使用的语音 |
| 语速 | 滑块 | 1.0 | 0.5-2.0 |

**高级设置**:
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 触发快捷键 | 快捷键 | Alt+T | 手动触发翻译 |
| 划词自动翻译 | 开关 | 开启 | 是否自动翻译划选内容 |
| 单词长度阈值 | 数字 | 20 | 单词/句子分界字符数 |

### 4.2 存储方案

使用Chrome Storage API:
- `chrome.storage.sync`: 用户配置（跨设备同步）
- `chrome.storage.local`: 缓存数据（翻译结果缓存）

**缓存策略**:
- **最大条目数**: 1000条
- **过期时间**: 7天
- **淘汰策略**: LRU (Least Recently Used)
- **键名格式**: `cache:{md5(text+sourceLang+targetLang)}`

## 5. 错误处理

### 5.1 错误类型

| 错误 | 处理方式 |
|------|----------|
| 引擎连接失败 | 显示错误提示，提供检查配置按钮 |
| 模型不存在 | 提示用户检查模型名称 |
| 翻译超时 | 显示"翻译超时，请重试" |
| 网络错误 | 显示"无法连接到本地模型服务" |
| 浏览器不支持TTS | 隐藏朗读按钮 |

### 5.2 用户反馈

- 翻译中显示加载动画
- 错误信息使用友好的中文提示
- 提供"重新翻译"按钮

### A. API请求示例

**标准接口** (OpenAI兼容格式，所有引擎通用):
```json
POST http://localhost:11434/v1/chat/completions
{
  "model": "qwen2.5:7b",
  "messages": [
    {"role": "system", "content": "你是翻译助手"},
    {"role": "user", "content": "翻译：hello"}
  ],
  "temperature": 0.3
}
```

**Ollama原生接口** (仅作为fallback):
```json
POST http://localhost:11434/api/generate
{
  "model": "qwen2.5:7b",
  "prompt": "翻译单词：hello",
  "stream": false
}
```

**API格式选择策略**:
1. 默认使用OpenAI兼容格式 (`/v1/chat/completions`)
2. 如果收到404错误且引擎为Ollama，自动fallback到原生格式
3. 适配器层封装差异，上层代码统一调用
